## **Invoice Center**

The [Invoice Center](https://bergendebate.retool.com/apps/4cb7af14-b658-11f0-b7e1-0f98c055d69f/Student%20Consolidation/Invoice%20Center%20With%20Zero%20Amount%20-SC) page allows admins to **generate and manage invoices** for students for a selected session and year. Admins first filter and select students, then choose how the payment will be collected.

### **Managing Selected Students**

Selected students on the **Add Student List** are added to the **Current Student List** on the right, where admins can review them and remove any student if needed before proceeding.

![Image](/images/image-88.png)

### Payment Methods Overview

![Image](/images/image-79.png)

**JotForm**\
Used when payment needs to be collected via an external JotForm link. The admin adds the JotForm URL and sends it to parents to complete the payment outside the platform.

**Stripe**\
Used for direct online payments. The admin sets the payable amount for card and/or bank transfer, which parents can pay securely through Stripe.

**Bucket**\
Used when the invoice amount is adjusted using internal credits or predefined pricing options, without requiring an immediate online checkout.

**Bucket Invoice Setup**

- Admin can apply **returner, sibling, or early-bird** options if applicable.
- Different amounts can be set for **card vs bank transfer**.
- If an invoice already exists, it can be **edited instead of recreated**.

The [Manage Invoice](https://bergendebate.retool.com/apps/8f8c2b60-6f39-11ef-81be-bf44285ea2ae/2024%20phase%202/Manage%20Invoice%20Q2-24) page lets admins view and manage all invoices in one place.\
Admins can search and filter invoices, check their processing status, and quickly **view, edit, or delete** invoices as needed.

## **Invoice List**

The[ Invoice List](https://bergendebate.retool.com/apps/53cebdf8-b65b-11f0-9b34-9fca107ca887/Student%20Consolidation/Invoice%20List%20Q2-24%20-%20SC) page allows admins to view and manage all student invoices in one place. Admins can quickly search, filter, and track invoice status, and take actions such as completing pending invoices.

![Image](/images/image-82.png)

### Key Fields:

- **Pre-registered**: Indicates whether the student was pre-registered before the invoice was generated.
- **Supplementary Program**: Shows if the invoice includes any additional or optional programs.
- **Status**: Displays the current state of the invoice (for example, processing, admin marked, or completed).
- **Actions**: Allows admins to **mark the invoice as complete**.

- **Prev Student (Self Reported)**\

- **Pre-Registrant**\
  Indicates if the student was pre-registered before the invoice was generated, which may affect pricing or discounts.
- **Supplementary Program**\

### Bucket Invoice vs Other Invoices

- **Bucket invoices** are manually assigned using **Add Bucket** and are generally used for structured or predefined pricing scenarios.
- Other invoices may be auto-generated during registration or checkout based on selected programs and discounts.

## Invoice Rates

The [Invoice Rates](https://bergendebate.retool.com/apps/0ac16b4a-ff24-11ef-b99e-67da19340c58/2025%20Phase%20B/Invoice%20Rates%20-%20V2) page allows admins to **define and manage invoice pricing rules (buckets)** for different years and sessions.

![Image](/images/image-85.png)

Admins can view existing **base pricing, fees, discounts, and deposit amounts**, and see how different combinations (returner, sibling, early bird) affect the final invoice amount.

**Create Bucket Popup**

Using **Create Bucket**, admins can set up a new pricing bucket for a selected year and session. If a bucket already exists, the system prevents duplication and prompts the admin to edit instead.

![Image](/images/image-83.png)

**Edit Bucket Popup**

Using Edit Bucket, admins can update existing pricing values. Any changes apply to future invoices generated for the selected year and session.

![Image](/images/image-84.png)
